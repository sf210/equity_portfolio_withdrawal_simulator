# Installers

One-step installers for the Equity Portfolio Withdrawal Simulator. Each one:

1. Checks for **Python 3.12 or newer** (with `tkinter`). If it is missing, it
   tells you and asks for permission before installing **Python 3.14**.
2. Creates a virtual environment in the project's `.venv/` directory.
3. Installs the dependencies from `requirements.txt` (NumPy, plus the optional
   docs-only `markdown`).
4. Creates a shortcut/app icon that launches the Monte Carlo GUI
   (`montecarlo_gui.py`).

The installers do **not** move the project — the app runs from this checkout, so
keep the folder where you want it before installing. They are safe to re-run;
the virtual environment and shortcuts are refreshed in place.

## Linux

```bash
installers/install_linux.sh
```

Adds an "Annuity Monte Carlo" entry to your applications menu and a Desktop
shortcut. If Python must be installed, the script uses your package manager
(`apt`, `dnf`, `zypper`, or `pacman`) and will ask for `sudo`. Removal:
`installers/uninstall_linux.sh`.

## macOS

Double-click **`install_macos.command`** in Finder (or run it from Terminal).
It builds `~/Applications/Annuity Monte Carlo.app`. If Python must be installed,
it uses Homebrew when present, otherwise the official python.org installer
(which bundles Tk). On the app's first launch, if Gatekeeper blocks it,
right-click the app and choose **Open**. Removal: `uninstall_macos.command`.

> If `3.14.0` is not the current 3.14 release when you run this, set the patch
> version: `PY_FULL=3.14.x ./install_macos.command`.

## Windows

Double-click **`install_windows.bat`** (it runs `install_windows.ps1` with the
right execution policy). It creates Start Menu and Desktop shortcuts that launch
the GUI with no console window. If Python must be installed, it downloads the
official python.org installer for your architecture and installs it per-user
with Tcl/Tk included. Removal:
`powershell -ExecutionPolicy Bypass -File installers\uninstall_windows.ps1`.

> If `3.14.0` is not current, set `PY_FULL` first, e.g.
> `set PY_FULL=3.14.x` then run the `.bat`.

## Icons

`icons/icon.png`, `icon.ico`, and `icon.icns` are generated by
`icons/make_icon.py` (run with a Python that has Pillow). Regenerate with:

```bash
python installers/icons/make_icon.py
```
